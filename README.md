# Практическая работа 3 Шаблоны проектов

Внутри архива два шаблона, один на C#, второй на Node.js.

Оба шаблона показывают работу с настройками из трёх источников, проверку корректности настроек до запуска, базовую защиту для запросов из браузера, ограничение частоты запросов и добавление защитных заголовков.

## Приоритет источников настроек

Порядок переопределения одинаковый в обоих шаблонах:

1. Файл настроек
2. Переменные окружения
3. Аргументы командной строки

## Критичные настройки

- `trustedOrigins` / `App:TrustedOrigins` - без списка доверенных источников служба не запускается.
- `rateLimits` / `App:RateLimits` - лимиты чтения и записи должны быть больше нуля, запись не должна быть выше чтения.
- `mode` / `App:Mode` - только учебный или боевой режим.

Ранняя проверка выполняется до запуска веб службы. При ошибках выводится понятная причина и запуск прерывается.

```powershell

```


```powershell

```

## Что проверяют тесты

- Приоритет источников настроек (файл -> окружение -> аргументы).

## Краткое описание проекта (для ответа преподавателю)

### Общая идея

Проект демонстрирует, как веб сервис читает настройки из трех источников, проверяет их корректность до запуска и применяет базовую безопасность: CORS для доверенных источников, ограничение частоты запросов и защитные заголовки. Есть два варианта реализации: C# (ASP.NET Core) и Node.js.

### Что делает C# приложение

1. Загружает настройки строго в порядке: файл `appsettings.json` -> переменные окружения (префикс `PR3_`) -> аргументы командной строки.
2. Валидирует критичные настройки до запуска. Если они некорректны, сервис печатает причины и завершает работу.
3. Регистрирует сервисы и in-memory репозиторий элементов.
4. Включает CORS только для доверенных источников.
5. Включает лимиты частоты для чтения и записи.
6. Добавляет защитные заголовки и централизованную обработку ошибок.
7. Поднимает API:
	- `GET /api/items` и `GET /api/items/by-id/{id}` для чтения.
	- `POST /api/items` для создания.
	- `GET /api/mode` для проверки режима работы.

### Что делает Node.js приложение

1. Читает настройки из файла, переменных окружения и аргументов (приоритет такой же).
2. Проверяет корректность настроек до запуска.
3. Настраивает CORS только для доверенных источников.
4. Включает ограничение частоты запросов.
5. Добавляет защитные заголовки.
6. Поднимает аналогичные маршруты API.

### Как устроены проверки

Тесты подтверждают, что:

- приоритет источников работает корректно;
- сервис не запускается при неправильных настройках;
- CORS разрешен только для доверенных источников;
- при превышении лимита возвращается 429;
- защитные заголовки присутствуют в ответе.
